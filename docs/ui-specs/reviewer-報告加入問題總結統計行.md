## 🔍 確認問題

### 1. 這個任務是否有 UI 層的變動？

你描述的需求核心是：

> 修改 Reviewer Agent 的**報告產出邏輯**，在報告末尾附加統計行

這聽起來是一個 **後端 / Agent 邏輯層**的修改（統計 emoji 出現次數、拼接字串），而非 UI 元件的新增或調整。

**請確認：**
- 報告輸出到 UI 的方式是否會改變（例如：現在是純文字串流、還是結構化資料）？
- UI 端是否需要針對這個統計行做**特殊渲染**（例如：獨立區塊、highlight、badge）？
- 還是統計行只是附加在報告文字末尾，UI 一律以現有方式顯示，**不需要任何 UI 層修改**？

---

### 2. 報告目前在 UI 上如何呈現？

我需要知道現有的呈現方式，才能判斷是否需要規格調整：

- 報告內容是透過 **SignalR streaming** 逐字輸出（像 ChatGPT 打字效果）？
- 還是一次性顯示完整 Markdown 字串？
- 顯示元件是 `MudBlazor` 的哪個元件（例如：自訂 Markdown renderer、`MudText`、`MudPaper` 內嵌文字）？

---

### 3. 統計行是否需要「置底固定」或「視覺上與報告本體區隔」？

- 如果統計行只是報告最後一行純文字 → **不需要 UI 規格，屬於 Agent 邏輯範疇**
- 如果統計行需要在 UI 呈現上與報告本體有所區別（例如：置底、加框、顯示為 summary badge）→ **才需要 UI 規格**
