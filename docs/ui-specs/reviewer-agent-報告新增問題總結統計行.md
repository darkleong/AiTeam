# UI 規格確認與提問

在開始產出規格文件之前，我需要先釐清幾個關鍵問題，確保規格範圍正確。

---

## 🔍 待確認事項

### 1. 此需求是否有 UI 呈現層面的變更？

從需求描述來看，這個任務的核心是：

> **修改 Reviewer Agent 的報告生成邏輯**（後端 / Agent 邏輯層）

產出的報告摘要行（`共發現 N 個 🔴 嚴重問題、M 個 🟡 建議`）是附加在**報告文字內容**的最末行，屬於純文字輸出的一部分。

這意味著：
- **主要變更在報告生成邏輯**，而非 UI 元件或頁面佈局
- UI 層（Blazor 頁面）只是**顯示**這段文字，本身不需要額外新增元件

### 2. 在我繼續產出之前，我需要你確認以下問題：

**Q1｜報告目前如何顯示？**
- 報告內容是以 Markdown 渲染呈現，還是純文字（`<pre>` / 純字串）？
- 還是透過 MudBlazor 的特定元件（如 `MudText`、自訂元件）顯示？

**Q2｜摘要行是否需要特殊 UI 處理？**
- 純粹是報告文字的一部分（不需要 UI 規格），還是需要在頁面上**特別凸顯**（例如：獨立的統計區塊、Badge、Chip 等元件）？

**Q3｜報告在哪個頁面顯示？**
- 是獨立的報告詳細頁？還是嵌在列表頁的展開區塊？
- 目前是否有既有的報告顯示 UI 可以參考？

## 💬 我的判斷

如果摘要只是**附加在報告文字末尾的一行純文字**，這個任務的核心是「Agent 報告生成邏輯修改」，**並不涉及 UI 規格設計的範疇**，不需要我產出 UI 規格文件。

如果需求是希望摘要統計**在 UI 層獨立呈現**（例如：固定顯示在報告卡片底部的統計列、可視化的嚴重度 Badge 等），那才需要 UI 規格介入。

**請確認你的預期是哪一種？** 我會依據你的回覆決定是否產出規格，以及規格的範疇。
